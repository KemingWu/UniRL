# EditReward scorer — vendored inference code under _editreward/.

# IMPORTANT: This file is intentionally EMPTY of pip packages. The reward
# service config validator requires a runtime_env file to exist, but Ray's
# runtime_env pip isolation is broken on the vllm-sglang-omni image (the
# created virtualenv lacks pip). Instead, the launcher script installs all
# dependencies directly into the main venv before starting the service.
#
# The vendored _editreward/ package needs only:
#   transformers, safetensors, omegaconf, qwen-vl-utils, torchvision,
#   numpy, packaging, requests, decord
# (training-only deps like fire/matplotlib/datasets/deepspeed are NOT needed
# because the vendored code strips all training imports.)
#
# Leaving this empty makes Ray create a bare venv (--system-site-packages)
# without attempting any pip install, so the main-env deps are used.
